### `[ 002 ]` — THE SECRET KEEPER

<table>
<tr>
<td width="60%">

**`REST_API_USING_SSS`** &nbsp;·&nbsp; ![JavaScript](https://img.shields.io/badge/JavaScript-F7DF1E?style=flat-square&logo=javascript&logoColor=black) ![Cryptography](https://img.shields.io/badge/Cryptography-Shamir_SSS-e63946?style=flat-square) ![REST](https://img.shields.io/badge/REST-API-00c896?style=flat-square)

> *1979 mathematics. 2024 API. Zero compromise.*

Shamir's Secret Sharing predates the internet. I turned it into a **production-grade REST API** — three endpoints that make secrets **mathematically unrecoverable** without a threshold of holders.

```
POST /split
  body: { secret, n, k }
  → Returns n unique polynomial shares

POST /reconstruct
  body: { shares[] }  ← must have ≥ k shares
  → Returns original secret via Lagrange interpolation

GET /info
  → API metadata
```

*Fewer than k shares = zero information. Not encrypted. Provably unknowable.*

</td>
<td width="40%" valign="top">

<br/><br/>

```
┌────────────────────────┐
│  THE MATH              │
├────────────────────────┤
│                        │
│  f(x) = secret +       │
│   a₁x + a₂x² + ...    │
│         + aₖ₋₁xᵏ⁻¹    │
│                        │
│  Each share = f(i)     │
│  Reconstruct via       │
│  Lagrange over GF(p)   │
│                        │
│  k-1 points → 0 info  │
│  k points   → ✅ done  │
└────────────────────────┘
```
